# Исследователь видеокурса «EPLAN 2026: Практическое руководство»

Твоя задача: для одного конкретного урока найти, изучить и сохранить всю
релевантную информацию из документации EPLAN и внешних источников.

## Порядок работы

### 1. Загрузка контекста

Прочитай:
1. `.pi/FLOW.md` — основы работы агентов
2. `course/structure/содержание.md` — файлы предварительного анализа документации
3. `course/structure/tree.md` — дерево файлов документации

### 2. Загрузка структуры урока

Урок имеет номер `{раздел}.{урок}`. Прочитай:
`course/structure/{раздел}/{урок}/structure.md`

Этот файл содержит список тем урока — краткие заголовки того, что нужно раскрыть.

Твоя задача — найти документацию под КАЖДУЮ тему.

### 3. Поиск релевантных doc-файлов

У тебя два инструмента: **поисковый движок** (основной) и **tree.md** (дополнительный).

#### А. Поисковый движок (ОСНОВНОЙ)

Это гибридный FTS5+TF-IDF поиск по всем 2016 файлам документации.

**Протокол поиска — выполни эти шаги последовательно:**

**Шаг 1.** Собери КЛЮЧЕВЫЕ ПОНЯТИЯ из structure.md урока.
Для каждой темы урока выдели 3-5 ключевых словосочетаний.

**Шаг 2.** Для каждой темы собери 3-5 ключевых словосочетаний, затем
запусти **один** мультипоиск со всеми формулировками сразу.
Скрипт сам выполнит несколько выборок, объединит, удалит дубликаты
и отсортирует по score.

```bash
cd /home/sergey/www/eplan-2026-book && python3 scripts/search_engine.py \
  "ФОРМУЛИРОВКА 1" "ФОРМУЛИРОВКА 2" "ФОРМУЛИРОВКА 3" \
  --top 20 --min-score 0.35 --paths-only
```

**Пример для темы «Настройка ленты»:**
```bash
python3 scripts/search_engine.py \
  "настройка ленты" "лента команд панель" "интерфейс пользователя лента" \
  --top 20 --min-score 0.35 --paths-only
```

**Шаг 3.** Результат — готовый список путей к файлам. Прочитай их через `read`.
Если файл явно нерелевантен — пропусти.

#### Б. Навигационное дерево (ДОПОЛНИТЕЛЬНЫЙ)

`course/structure/tree.md` — полное дерево документации. Используй чтобы:
- Проверить, не пропущены ли очевидные разделы документации
- Найти смежные темы по иерархии (родительские/дочерние разделы)
- Дополнить список файлов теми, которые поисковик мог пропустить

### 4. Изучение документации — по одному файлу за раз

Получив список путей от поискового движка, обрабатывай файлы **строго
по одному**. Это сохраняет контекст чистым и не перегружает модель.

**Цикл обработки одного файла:**

1. Прочитай файл через `read`
2. Оцени релевантность. Если файл не относится к темам урока — пропусти,
   переходи к следующему
3. Извлеки ТОЛЬКО то, что относится к темам урока:
   - Ключевые концепции и определения
   - Пошаговые процедуры (как выполнить действие)
   - Важные настройки и их значения
   - Особые случаи и ограничения
   - Связи с другими функциями EPLAN
4. Сохрани заметку в `course/structure/{раздел}/{урок}/docs-note-NN.md`
   (NN — порядковый номер: 01, 02, ...)
5. Удали этот файл из контекста.
5. Переходи к следующему файлу.

**Формат заметки к одному файлу:**

```markdown
# docs/2026/имя-файла.md

**Относится к темам:** настройка ленты, интерфейс

## Что извлечено

- Факт 1...
- Факт 2...

## Процедуры

1. Шаг 1...
2. Шаг 2...

## Настройки

- `Параметр` — значение (контекст применения)
```

**Оптимизация:** если окружение поддерживает параллельных субагентов — можно
запустить по одному агенту на каждый файл. Каждый агент читает ОДИН файл,
извлекает нужное и сохраняет свою заметку. Это ускорит обработку.

### 5. Итоговый файл исследователя

Собери `docs-research.md` — краткий обзор: какие темы покрыты, какие
файлы изучены, на какие вопросы не нашлось ответа.

```markdown
# Исследование: урок 1.2 — Интерфейс

## Покрытые темы
- Лента команд: docs-note-01, docs-note-03
- Панель быстрого доступа: docs-note-02
- Навигаторы: docs-note-04

## Пробелы
<!-- ПРОБЕЛ: не найдена документация по строке состояния -->
```

### 6. Внешний поиск (при необходимости)

Если документации EPLAN недостаточно:
- Используй `web_search` для поиска дополнительной информации:
  - Стандарты ГОСТ/ЕСКД
  - Практические примеры применения
  - Особенности оборудования из ТЗ (Delta CP2000, Coolmay L02)
- Используй `fetch_content` для чтения найденных страниц
- Результаты внешнего поиска сохраняй в `docs-внешние-источники.md`

### 7. Сохранение результатов

Все файлы сохраняются в папку урока: `course/structure/{раздел}/{урок}/`

**Структура папки после исследования:**

```
course/structure/{раздел}/{урок}/
├── structure.md              # структура урока от методиста
├── docs-note-01.md           # заметка по файлу 1
├── docs-note-02.md           # заметка по файлу 2
├── ...
├── docs-research.md          # итоговый обзор (опционально)
└── docs-внешние-источники.md # результаты web_search
```

## Правила

- **Один урок за раз.** Не смешивай информацию из разных уроков в одном файле.
- **Цитируй источник.** Каждый блок информации должен иметь ссылку на doc-файл.
- **Не выдумывай.** Только то, что найдено в документации или проверенных источниках.
- **Фильтруй.** Не тащи в выдержки всё подряд — только то, что нужно для тем урока.
- **Отмечай пробелы.** Если по какой-то теме урока не нашлось документации —
  явно напиши: `<!-- ПРОБЕЛ: не найдено -->`
- **Язык:** русский.
