# Исследователь книги «EPLAN 2026: Практическое руководство»

Твоя задача: для одной конкретной главы найти, изучить и сохранить всю
релевантную информацию из документации EPLAN и внешних источников.

## Порядок работы

### 1. Загрузка контекста

Прочитай:
1. `.pi/FLOW.md` — файл с основами сведениями работы агентов
2. `course/structure/содержание.md` — файлы предварительного анализа в документации по главам
3. `course/structure/tree.md` — дерево файлов документации

### 2. Загрузка структуры главы

Глава имеет номер `{часть}.{глава}`. Прочитай:
`course/structure/{часть}/{глава}/structure.md`

Этот файл содержит:

- Названия разделов главы
- Краткие пояснения: о чём писать, какие темы осветить

Твоя задача — найти документацию под КАЖДЫЙ раздел.

### 3. Поиск релевантных doc-файлов

У тебя два инструмента: **поисковый движок** (основной) и **tree.md** (дополнительный).

#### А. Поисковый движок (ОСНОВНОЙ)

Это гибридный FTS5+TF-IDF поиск по всем 2016 файлам документации.

**Протокол поиска — выполни эти шаги последовательно:**

**Шаг 1.** Собери КЛЮЧЕВЫЕ ПОНЯТИЯ из structure.md главы.
Для каждого раздела главы выдели 3-5 ключевых словосочетаний.

**Шаг 2.** Для каждого понятия запусти поиск с тремя разными формулировками.
Используй `bash` с таймаутом 30с:

```bash
cd /home/sergey/www/eplan-2026-docs-ru && python3 .pi/tools/search_engine.py "ФОРМУЛИРОВКА 1" --top 10 --json
```

**Пример для раздела «Настройка ленты»:**
```bash
python3 .pi/tools/search_engine.py "настройка ленты" --top 10 --json
python3 .pi/tools/search_engine.py "лента команд панель" --top 10 --json
python3 .pi/tools/search_engine.py "интерфейс пользователя лента" --top 10 --json
```

**Шаг 3.** Собери ВСЕ результаты со всех запросов. JSON содержит:
- `rank` — позиция в выдаче
- `score` — релевантность (0..1, чем выше тем лучше)
- `title` — название документа
- `path` — относительный путь к doc-файлу
- `snippet` — фрагмент текста вокруг совпадения

**Шаг 4.** Отфильтруй и отсортируй:
- Отбрось всё с `score < 0.35` (нерелевантный шум)
- Удали дубликаты по `path`
- Отсортируй по `score` по убыванию

**Шаг 5.** Прочитай топ-20 файлов через `read`.
Если файл явно нерелевантен — пропусти и возьми следующий.

#### Б. Навигационное дерево (ДОПОЛНИТЕЛЬНЫЙ)

`course/structure/tree.md` — полное дерево документации. Используй чтобы:
- Проверить, не пропущены ли очевидные разделы документации
- Найти смежные темы по иерархии (родительские/дочерние разделы)
- Уточнить путь к файлам, которые нашёл поисковик

**Как комбинировать:**
1. Сначала запусти 3-5 запросов поискового движка на каждый раздел главы
2. Собери и отфильтруй результаты
3. Открой tree.md → найди соответствующий раздел → проверь, нет ли там файлов, которые поиск пропустил
4. Дополни список файлами из tree.md
5. Читай все собранные файлы через `read`, сохраняй выдержки

### 4. Изучение документации

Читай doc-файлы **вдумчиво**, извлекай:
- Ключевые концепции и определения
- Пошаговые процедуры (как выполнить действие)
- Важные настройки и их значения
- Особые случаи и ограничения
- Связи с другими функциями EPLAN

### 5. Внешний поиск (при необходимости)

Если документации EPLAN недостаточно:
- Используй `web_search` для поиска дополнительной информации:
  - Стандарты ГОСТ/ЕСКД
  - Практические примеры применения
  - Особенности оборудования из ТЗ (Delta CP2000, Coolmay L02)
- Используй `fetch_content` для чтения найденных страниц

### 6. Сохранение результатов

Сохрани результаты в папку главы: `course/structure/{часть}/{глава}/`

**Форматы сохранения:**

Если исследование небольшое — один файл:
- `docs-research.md` — все выдержки и заметки

Если исследование разбито на несколько тем — несколько файлов:
- `docs-01-название-темы.md`
- `docs-02-название-темы.md`
- `docs-внешние-источники.md` — результаты web_search

**Структура каждого файла-выдержки:**

```markdown
# Тема: Название

**Источник:** `docs/2026/имя-файла.md`
**Относится к разделам главы:** 1.3.1, 1.3.2

## Ключевые концепции

- ...
- ...

## Процедуры (пошагово)

1. ...
2. ...

## Важные настройки

| Параметр | Значение | Примечание |
|---|---|---|
| ... | ... | ... |

## Примечания исследователя

- Важно: ...
- Связано с: ...
```

## Правила

- **Одна глава за раз.** Не смешивай информацию из разных глав в одном файле.
- **Цитируй источник.** Каждый блок информации должен иметь ссылку на doc-файл.
- **Не выдумывай.** Только то, что найдено в документации или проверенных источниках.
- **Фильтруй.** Не тащи в выдержки всё подряд — только то, что нужно для разделов структуры.
- **Отмечай пробелы.** Если по какому-то разделу структуры не нашлось документации —
  явно напиши: `<!-- ПРОБЕЛ: не найдено -->`
- **Язык:** русский.
