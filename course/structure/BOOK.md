# EPLAN 2026: Практическое руководство — Базовый файл проекта

## Назначение этого файла

Этот файл загружается в начале **каждой сессии работы над книгой**. Он содержит:
- концепцию и цель книги;
- техническое задание (ТЗ), которое является сквозным проектом книги;
- стратегию написания глав;
- текущий статус работы над каждой главой.

Книга пишется как **пошаговое выполнение технического задания** из файла `book/TZ.md`.

---

## Техническое задание (сквозной проект)

**Файл ТЗ:** `book/TZ.md`

Проект насосной станции поддержания давления:
- 2 насосных агрегата (основной + резервный)
- Частотно-регулируемый привод: **Delta CP2000**
- ПЛК: **Coolmay L02**
- Автоматический режим (ПИД-регулирование по давлению, ротация, каскад) + ручной
- Связь ПЛК ↔ ЧРП: Modbus RTU (чтение данных)
- Стандарты документации: **ГОСТ, ЕСКД**

Каждая глава книги должна явно показывать, **как конкретный инструмент EPLAN применяется для реализации этого ТЗ**.

## Стратегия написания глав

### Принцип «ТЗ как нить»

Книга не является справочником по EPLAN — она является **дневником проекта**. Читатель вместе с автором проходит весь путь от пустого проекта до готового комплекта документации. Каждая глава открывает новые возможности EPLAN **именно тогда, когда они нужны для следующего шага ТЗ**.

### Структура каждой главы

1. **Введение к главе** — что мы делаем в этой главе и зачем это нужно для нашей насосной станции.
2. **Теория** — минимально необходимые концепции EPLAN (опираемся на документацию из `docs/eplan/`).
3. **Практика шаг за шагом** — конкретные действия в интерфейсе, скриншот за скриншотом.
4. **Результат** — что получилось, как это выглядит в проекте насосной станции.
5. **Итог главы** — краткое резюме и переход к следующей главе.

### Правила написания

- **Каждый шаг требует скриншота.** Вставлять плейсхолдер: `![Описание скриншота](images/НомерГлавы_НомерШага.png)`
- **Клавиши** в формате pymdownx.keys: `++ctrl+s++`, `++alt++`, `++enter++`
- **Предупреждения и советы** оформлять как admonition: `!!! tip`, `!!! warning`, `!!! note`
- **Все настройки проекта** — строго по ГОСТ и ЕСКД
- **Аббревиатуры** подхватываются из `docs/includes/abbreviations.md`

### Источники (RAG)

Перед написанием каждой главы читать соответствующие файлы документации из `docs/eplan/`, указанные в `book/содержание.md` для данного раздела.

## Процесс работы над главой (для AI-сессий)

При старте сессии:
1. Прочитать этот файл (`book/BOOK.md`) — контекст проекта.
2. Прочитать `book/TZ.md` — сквозное техническое задание.
3. Определить текущую главу по статусу ниже (первая со статусом `🔄` или `⬜`).
4. Прочитать соответствующие файлы документации из `docs/eplan/` согласно `book/содержание.md`.
5. Написать главу согласно структуре выше.
6. Сохранить главу в `book/chapters/ЧастьX_НазваниеГлавы.md`.
7. Обновить статус в этом файле.
8.
